# 《设计理论随身背》内容维护说明

本文说明：**不写代码的前提下**，日常要在哪些文件里改字、加条目，以及改完怎么在电脑上看到效果。  
当前示例书籍是《设计学概论》第三版，内容主要放在 `src/data/designIntro/` 文件夹里。

---

## 1. 各类内容分别在哪些文件里维护？

| 内容类型 | 主要维护文件 | 说明 |
|---------|-------------|------|
| **书库列表**（书名、作者、封面说明等） | `src/data/books.js` | 在书库里显示的那几本书在这里登记。 |
| **某一本书的章节目录、章节介绍** | `src/data/designIntro/chapters.js` | 每一章的标题、摘要、逻辑框架等。 |
| **题库**（名词解释题、简答题、论述题等） | `src/data/designIntro/questions.js` | 题目正文、答题要点、推荐模板等。 |
| **名词解释**（知识点卡片） | `src/data/designIntro/terms.js`（会合并 `termsCh1.js` 等） | 第一章扩展词条在 `termsCh1.js`，其余章节示例在 `terms.js` 里的「后半段列表」。新增时二选一：跟第一章放一起就改 `termsCh1.js`；其他章节可仿照 `terms.js` 里现有条目追加。 |
| **答题模板** | `src/data/designIntro/templates.js` | 各类题型的写作骨架。 |
| **关键词、思维导图、时间线、大纲树** | `src/data/designIntro/keywords.js`、`mindmaps.js`、`timelines.js`、`outline.js` | 按文件名对应功能。 |
| **PDF 摘录备忘（不显示在页面上）** | `src/data/designIntro/pdfSourceNotes.js` | 只给自己对照用，用户界面不会读取展示。 |
| **金句、会员说明、用户身份键名** | `src/data/warmQuotes.js`、`membershipPlans.js`、`userMock.js` | 一般很少改。 |

**补充：** `src/data/catalog.js` 像一本「总目录」，把上面这些模块挂到具体的书号上。现阶段只有《设计学概论》这一本在目录里；将来若增加新书，需要程序员在 `catalog.js` 里多挂一条（普通维护内容时可先不改这里）。

## 2. 如何新增一个名词解释？

1. 打开 `src/data/designIntro/terms.js`，看文件前半部分：第一章很多条目来自 `termsCh1.js`。  
   - 若你的词条属于**第一章**，建议改 **`termsCh1.js`**，保持第一章集中维护。  
   - 若属于**其他章节**，在 `terms.js` 里 **`designIntroTermsRest` 那个数组**中，照着已有条目**复制一整段**，再改字。
2. 每一条必须有一个**全项目唯一**的 `id`（建议格式：`t-章节简称-词条英文名或拼音`，例如 `t-ch2-xxx`），**不要和已有 id 重复**。
3. `chapterId` 必须和 `chapters.js` 里某一章的 `id` 一致（例如 `ch1`、`ch2`），否则章节页里找不到这条。
4. `bookId` 写 `design-introduction-3`（与 `chapters.js` 顶部的书号一致即可；若从现有条目复制，一般不用改）。
5. 填好 `title`、`keywords`、`shortAnswer`、`expandedAnswer`、`answerStructure`、`memoryTip` 等；可选填 `relatedQuestionIds`（对应题库里的题目 id）、`relatedTemplateIds`（模板 id）。
6. 保存文件后，用下面第 7 节的方式本地预览，在 App 里从**书库 → 该书 → 章节 → 名词解释**点进去检查。

---

## 3. 如何新增一道简答题？

1. 打开 `src/data/designIntro/questions.js`。
2. 在 `designIntroQuestions` 这个**数组**里，**复制一道已有的简答题**整块（找 `typeKey: "short"` 的示例）。
3. 修改：  
   - `id`：改成新的、全项目唯一（如 `q-s99`）。  
   - `title`：题干。  
   - `chapterId`：归属章节，要和 `chapters.js` 一致。  
   - `typeKey`：**简答题必须写 `"short"`**。  
   - `compact`、`structure`、`expanded`、`answerIdea`、`keywords` 等按需要改。
4. 保存后本地预览，在**题库**里筛选简答题，或从章节关联入口查看是否出现。

---

## 4. 如何新增一道论述题？

步骤与简答题相同，都在 `questions.js` 里操作。区别只有一点：

- `typeKey` 写成 **`"essay"`**（表示论述题）。

其余字段照样参考文件里已有的论述题条目修改即可。

---

## 5. `shortAnswer` 和 `expandedAnswer` 应该怎么写？

这两个字段出现在**名词解释词条**里（不是题库里的 `compact` / `expanded`，那是题目用的名字，但写法思路类似）。

- **`shortAnswer`（精简背诵版）**  
  - 给同学**直接背**用的，不要只写一句话带过。  
  - 语气偏**考研答卷**：完整句、有层次，少用聊天口吻。  
  - 需要强调的词，可以用英文半角符号包起来：`**关键词**`（页面上会高亮）。  
  - 篇幅大致参考：**名词解释类**约 120～180 字；**简答题类**约 180～260 字；**论述题类**约 250～350 字（不必卡死，接近即可）。

- **`expandedAnswer`（展开论述版）**  
  - 比精简版**更完整**，适合考场上**展开写**。  
  - 建议心里按四块组织：**概念解释 → 主要内容 → 影响或意义 → 考试可用的一句话收束**；需要时可加例子、人物、流派。  
  - 篇幅大致参考：**名词解释类**约 250～400 字；**简答题类**约 400～600 字；**论述题类**约 600～900 字。  
  - 尽量用「自己的整理语言」，**不要大段粘贴教材或 PDF 原文**（避免版权与雷同）；PDF 可摘录到 `pdfSourceNotes.js` 里自己对照。

---

## 6. `accessLevel` 是什么意思？

名词解释条目里有一个字段 **`accessLevel`**，用来表示「谁有权看完整版」：

| 取值 | 含义 |
|------|------|
| **`free`** | 免费用户也能看**全文**（适合少量引流、建立信任的条目）。 |
| **`premium`** | 需要**高级会员**身份才能看完整精简+展开+结构等；免费用户可能只看到前面一小段预览。 |
| **`coaching`** | 需要**陪跑学员**身份（比高级会员更高一档）；用于将来押题、院校专项等更靠服务的部分。 |

当前身份是在浏览器里**本地模拟**的（会员页底部可切换），不接真实支付。未写 `accessLevel` 的条目，程序里会按**免费**处理。

## 7. 修改内容后如何本地预览？

1. 在电脑上打开终端（命令行），进入本项目文件夹（包含 `package.json` 的那一层）。  
2. 第一次或依赖有变化时执行：`npm install`  
3. 启动开发预览：`npm run dev`  
4. 终端里会出现一个本地地址（一般是 `http://localhost:5173` 之类），用浏览器打开即可实时看到修改效果。  
5. 改完保存文件，浏览器里**刷新页面**或按项目提示热更新即可。

若只想确认能否正常打包，可执行：`npm run build`（成功表示语法结构大致没问题）。

## 8. 常见报错原因（非程序员版）

改数据文件时，最常见的问题是**标点或括号少写、多写**，导致整页白屏或构建失败。可以对照检查：

1. **少逗号**  
   每一条属性之间要用**英文逗号** `,` 隔开（不能用中文逗号）。  
   上一条的末尾 `}` 后面，若还有下一条，**必须有逗号**。

2. **引号成对**  
   文字要用**英文双引号** `"……"` 包起来；里面的内容若出现双引号，要么改成单引号叙述，要么用 `\"` 转义（不熟悉时尽量少在正文里用英文双引号）。

3. **数组格式**  
   `keywords`、`answerStructure`、`relatedTemplateIds` 等是**数组**：以 `[` 开头、`]` 结尾，每一项之间用英文逗号。最后一项后面**不要多逗号**（部分环境会报错）。

4. **大括号要配对**  
   每一条名词解释或题目是一整段 `{ … }`；少写一个 `}` 会导致后面全部解析失败。

5. **复制时带了中文标点**  
   如 `：`、`，`、`「` 若误用在程序结构位置可能没事，但**作为代码分隔符**必须用英文符号。保险做法：**只改中文正文，不要改外层的 `[]` `{}` `: ,` 结构**。

6. **id 重复**  
   两条内容用了同一个 `id`，后一条会覆盖或行为异常；新增时务必换一个**全新 id**。

若页面突然空白，可打开浏览器的**开发者工具 → Console（控制台）**，看红色报错的第一行，往往指向「第几行附近少了逗号或括号」。
